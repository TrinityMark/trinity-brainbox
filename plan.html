🧠 Trinity BrainBox — Build Plan

Local web app for Trinity Advisory staff · Plain HTML/JS · No server required

Why we're building this

Trinity Advisory staff need a quick interface to pull together client context and curated prompts before jumping into their custom ChatGPT. Instead of hunting through SharePoint folders and copy-pasting manually, Trinity BrainBox gives them a guided three-step flow.

1

Pick a client (or work without one)

2

Grab the client's files from SharePoint, copy names for upload to GPT

3

Select a prompt — client name auto-fills — copy it straight into GPT

Files we'll create
index.html

The entire app — HTML, CSS, and JavaScript all in one file. Open directly in Chrome or Edge, no server needed.

config.js

Holds the three Power Automate webhook URLs. The only file staff need to update when flows change.

Power Automate Flows (3 total)
EXISTS

Flow 1 — Get Clients

Reuse the existing flow from the coaching dashboard project. HTTP POST → returns [{ "name": "Acme Corp" }, ...]. Copy its webhook URL into config.js.

TO BUILD

Flow 2 — Get Client Files

HTTP POST with body { "clientName": "Acme Corp" }. Power Automate queries SharePoint Shared Documents/{clientName}/ClientBrain and returns the file list + folder URL.

Site: https://trinityaccountants.sharepoint.com/sites/TrinityClients

TO BUILD

Flow 3 — Get Prompts

HTTP POST (no body). Power Automate reads all .txt files from a SharePoint prompts folder and returns [{ "name": "...", "content": "..." }, ...]. Start flat — categories can be added later.

Prompt Placeholder System

Anyone writing prompts in SharePoint uses double-curly-brace syntax. The app replaces these automatically when a prompt is selected.

Placeholder Replaced with If no client selected
{{ClientName}} The selected client's name Shows [Client Name] as a visual cue
{{FileList}} Newline-separated list of selected file names Shows [File List] as a visual cue
Build Order
1

Create config.js

Three placeholder webhook URLs with comments explaining what each flow must return. This is where staff paste real URLs once flows are built.

2

Build index.html — structure and styling

Clean, professional look. White background, dark teal (#0d4f5c) accents, sans-serif. Three-section layout matching the wireframe above.

3

Wire up client dropdown (Flow 1)

On page load, POST to Flow 1 and populate the dropdown. "No specific client" always appears at the top.

4

Wire up client files section (Flow 2)

On client selection, call Flow 2, show the SharePoint folder URL with a copy button, list files as checkboxes, and show a "copy selected file names" button.

5

Wire up prompts (Flow 3)

Load prompts on page load in parallel with Flow 1. Show as clickable cards. On click, display text in preview panel with placeholders substituted. Copy button copies filled-in text.

6

Error states and polish

If a flow URL isn't configured, show a clear "not yet configured" notice instead of a broken request. Loading spinners while flows run.
