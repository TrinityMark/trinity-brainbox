Power Automate Flows

Webhook URLs are stored in config.js in the project root. This file is gitignored because the URLs contain signed access tokens -- copy config.sample.js to config.js and fill in the real URLs after cloning. Update config.js when flows are created or changed.

Data Flow

1. Browser opens index.html locally.
2. On load, fetch() calls Flow 1 (clients) and Flow 3 (prompts) in parallel.
3. Client dropdown and prompt list are populated from flow responses.
4. When a client is selected, fetch() calls Flow 2 with the client name in the POST body.
5. File list and folder path are displayed from Flow 2 response.
6. When user selects files and a prompt, placeholders are replaced in the browser -- no further server call.
7. Copy buttons use the Clipboard API to place the result on the user's clipboard.

Network and Security Notes

The app makes outbound HTTPS POST requests to Power Automate webhook URLs (Microsoft-hosted). No client data is stored by the app -- all data is fetched live and held in browser memory only. The app does not transmit data to any third parties. Only file names and the client name are passed to flows; file content is never sent. The app should only be opened on Trinity Advisory staff machines with standard Microsoft 365 credentials.

Dependencies

None. The app has zero external dependencies -- no CDN links, no npm packages, no frameworks.
